Crisis Cascades

How platform capitalism creates compounding system failures that accelerate institutional collapse.

Crisis Cascades describe how platform-driven disruption creates compounding system failures. Unlike traditional market competition, platforms actively degrade institutional capacity and social infrastructure, making each subsequent crisis more likely and more severe.

Crisis Cascade Mechanisms

Capacity Stripping

The systematic degradation of institutional ability to function independently through technical systems that convert direct capabilities into platform-dependent services.

Network Resilience Degradation

Breaking down informal support networks and social infrastructure that enable communities to respond to crisis and maintain stability.

Platform Parasitism

The process by which platforms actively degrade systems' ability to function independently, creating extractable dependencies that enable further degradation.

Crisis Acceleration

How each system failure makes subsequent crises more likely and more severe by undermining institutional and community resilience.

Related Framework Concepts

Interface Gradient Theory

How platforms create progressively more abstract layers between users and functions, with each layer becoming a new point of control and extraction. As systems become more 'advanced', they:

  • Create additional layers between users and functions
  • Turn each layer into a subscription/control point
  • Make direct interaction increasingly difficult
  • Convert straightforward tasks into platform-mediated experiences

Manufactured Chaos

The deliberate creation of systemic disruption to degrade institutional capacity and force dependency through:

  • Interface Disruption - Suddenly disabling critical systems
  • Institutional Degradation - Creating paralyzing uncertainty
  • Cascading Effects - Triggering system-wide failures
  • Support Network Disruption - Overwhelming community resources

Friction Engineering

How platforms deliberately create and manipulate friction points to enable extraction and control:

  • Degradation Phase - Reduce direct service quality
  • Problem Definition Phase - Frame degradation as external
  • Platform Solution Phase - Introduce platform requirements
  • Extraction Evolution - Convert basic functions to "features"

Constitutional Interface Degradation

How technical systems deliberately obscure and override legal rights and constitutional obligations:

  • Technical systems overriding legal frameworks
  • Constitutional obligations converted to technical privileges
  • Rights becoming platform-mediated
  • Infrastructure designed for political control

Memory Markets

How financialization commodifies institutional knowledge and expertise:

  • Knowledge networks as financial assets
  • Expertise as tradable commodity
  • Institutional memory as property
  • Learning systems as capital
